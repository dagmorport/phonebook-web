## Версия 5.5 — Исправление авторизации

### Критические исправления
- **Баг scope в PowerShell event handlers**: Переменная `$result` внутри `Add_Click` создавала локальную копию — внешний `$result` оставался `$false`. Диалог входа всегда возвращал `false` → `exit` → главное окно не появлялось. Исправлено: `$script:loginResult` / `$script:adminDialogResult`
- **DispatcherTimer GC**: Таймер `Invoke-Refresh` создавался внутри event handler как локальная переменная — сборщик мусора уничтожал его. Таймер пересоздавался → бесконечный цикл загрузки. Исправлено: хранение в `$script:refreshTimer`

### Аутентификация (стиль Export-Phonebook.ps1)
- Проверка группы `Domain Admins` через `whoami /groups` (локально, без LDAP)
- Fallback: `memberOf` через `DirectorySearcher` (одно подключение к DC)
- Убран `GetGroups()` — создавал отдельное LDAP-подключение к غير рабочему `srv-dc-001`
- Все подключения к AD идут напрямую к `srv-dc-002.e5dag.ru` (без auto-discovery)
- `Get-ADUsers` подключается через `DirectoryEntry("LDAP://srv-dc-002")` с токеном текущей сессии

### Улучшения
- `Show-ADErrorMessage` теперь показывает реальное сообщение ошибки вместо generic «Обратитесь к администратору»
- Статус-бар отображает текст ошибки при проблемах с AD
- Таймер загрузки данных: 800ms задержка перед `Invoke-Refresh` (окно успевает отрисоваться)
- Обработка ошибок `Invoke-Refresh` через try-catch в DispatcherTimer (не крашит приложение)

## Phonebook.ps1 — Исправления безопасности

### Критические
- LDAP-инъекция: санитизация CN при создании пользователя
- Валидация email, телефона, длины полей при записи в AD
- Установка пароля при создании пользователя (RandomNumberGenerator)
- sAMAccountName: проверка длины (≤20), пустого имени, ASCII

### Высокие
- Сообщения об ошибках: `$_.Exception.Message` не показывается пользователю
- Функция `Show-ADErrorMessage` — логирует в консоль, показывает общее сообщение

### Средние
- `Set-ADAttribute` вынесена на верхний уровень
- `Test-DomainAdmin`: finally-блок для Dispose ресурсов
- `Sanitize-ADString`: фильтрация LDAP-опасных символов

### Аутентификация
- Диалог пароля для Domain Admin (`$xamlPasswordDialog`)
- `Test-AdminPassword`: LDAP bind + проверка группы Domain Admins
- `$script:isEditUnlocked` — разблокировка на сессию

## Export-Phonebook.ps1 — Аутентификация

### Что добавлено
- **`Get-Credential`** — запрос логина/пароля при запуске bat
- **LDAP-проверка** — `PrincipalContext.ValidateCredentials()` на `srv-dc-002.e5dag.ru`
- **Проверка Domain Admins** — `UserPrincipal.GetGroups()` → проверка «Domain Admins»
- При ошибке — красное сообщение + `exit 1`
- При отмене — `exit 1`

### Правила безопасности
- Только Domain Admins могут запускать экспорт
- Пароль не сохраняется — используется только для проверки
- Обычные сотрудники не допускаются

### Исправления
- `GetGroups()` заменён на `whoami /groups` — проверка группы Domain Admins локально, без доп. подключения к DC (ошибка `srv-dc-001 not operational`)
- Формат логина: `Get-Credential` → чистый `username` для `ValidateCredentials`
